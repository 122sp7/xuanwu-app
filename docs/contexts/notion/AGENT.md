# Notion Agent

## Mission

保護 notion 主域作為核心知識內容生命週期邊界。任何變更都必須維持 notion 對知識內容的正典所有權，不把平台治理或 notebooklm 對話語言直接混入內容模型。

## Owns

- knowledge
- authoring
- collaboration
- database
- ai
- analytics
- attachments
- automation
- integration
- notes
- templates
- versioning

## Route Here When

- 問題是 Knowledge Page、Article、Database、Template、Attachment 或 Version Snapshot。
- 問題是知識內容的編輯、分類、協作、結構化視圖或內容自動化。
- 問題需要把外部輸入轉化為正典知識內容。

## Route Elsewhere When

- 身份、帳號、組織、授權、政策、方案與通知屬於 platform。
- workspace 容器、活動流、排程與工作區流程屬於 workspace。
- notebook、conversation、source、note、synthesis 屬於 notebooklm。

## Working Rules

- 先保護知識內容正典語言，再接入外部能力。
- AI 輔助可以服務 notion，但 AI 對話語言本身不直接成為 notion 的正典模型。
- 跨主域協作只透過 published language、事件或 API 邊界。
- 新增跨子域術語時，先更新 [ubiquitous-language.md](./ubiquitous-language.md)。

## Documentation Checklist

- 子域變更時，同步更新 [subdomains.md](./subdomains.md)。
- 邊界變更時，同步更新 [bounded-contexts.md](./bounded-contexts.md) 與 [context-map.md](./context-map.md)。
- 新增或調整跨子域術語時，同步更新 [ubiquitous-language.md](./ubiquitous-language.md)。
