# Wiki 使用手冊

> **文件版本**：v1.0.0
> **最後更新**：2026-03-20
> **目標讀者**：工作區成員、組織管理員、組織擁有者

---

## 概覽

Wiki 是 xuanwu-app 的**現代型知識中樞**，整合了組織知識庫與工作區知識，讓您可以：

- 📚 在一個地方瀏覽**整個組織的企業知識**
- 🗂️ 查看各工作區的**文件與知識文件狀態**
- 🔍 使用 **AI 語意搜尋**（RAG）直接向知識庫提問
- 📝 撰寫並共用 **Wiki 頁面**（操作說明、專案紀錄、政策文件等）
- 🔒 管理私人頁面，僅自己可見

---

## 進入 Wiki

1. 登入後，在左側**側邊欄**點擊 **「Wiki」** 圖示
2. 系統會開啟 Wiki 主頁，左側顯示 Wiki 側邊欄導覽

---

## Wiki 側邊欄導覽

Wiki 側邊欄分為五個區塊：

| 區塊 | 說明 |
|------|------|
| 📌 **首頁** | Wiki 的起始主頁 |
| 🏢 **組織知識庫** | 企業級跨工作區知識文件（RAG 可搜尋） |
| 🗂️ **工作區知識** | 各工作區上傳並索引的知識文件 |
| 📝 **Wiki 頁面** | 共用與私人 Wiki 頁面 |
| 🗑️ **封存** | 已封存的頁面（預設折疊） |

## 組織知識庫

### 什麼是組織知識庫？

組織知識庫收錄了**所有工作區**已上傳並完成 AI 索引（`status: ready`）的知識文件。這些文件可跨工作區統一搜尋，是企業知識管理的核心。

### 如何瀏覽組織知識庫？

1. 點擊側邊欄的 **「組織知識庫」**
2. 您可以看到：
   - 📊 **知識庫總覽**：文件總數、已就緒數量、處理中數量
   - 🏷️ **分類瀏覽**：依 Taxonomy 分類（例如：規章制度、技術文件、產品手冊）
   - 🔍 **AI 搜尋入口**：向整個知識庫提問

### 使用 AI 搜尋

1. 在組織知識庫區塊頂部，找到搜尋框
2. 輸入您的問題，例如：
   - 「員工特休天數如何計算？」
   - 「系統上線的標準流程是什麼？」
   - 「哪個工作區有最新的安全規範文件？」
3. 按下 Enter 或點擊搜尋按鈕
4. 系統會透過 AI 分析知識庫內容並生成回答，同時附上**引用來源**（文件名稱、頁碼）

> **提示**：問題描述越具體，AI 回答越精準。

## 工作區知識

### 什麼是工作區知識？

工作區知識顯示**特定工作區**已上傳並索引的知識文件。每個工作區的文件僅在該工作區範疇內搜尋（除非特別標記為組織級共用）。

### 如何查看工作區知識？

1. 點擊側邊欄的 **「工作區知識」**
2. 選擇您想查看的工作區
3. 您可以看到：
   - 文件清單（包含 MIME 類型、大小、狀態、分類）
   - 各文件的處理狀態（見下方狀態說明）

### 文件狀態說明

| 狀態 | 圖示 | 說明 |
|------|------|------|
| 已上傳 (`uploaded`) | 🔵 | 檔案已上傳，等待 AI 處理 |
| 處理中 (`processing`) | 🟡 | AI 正在解析、切分並建立向量索引 |
| 已就緒 (`ready`) | 🟢 | 已完整索引，可供 AI 搜尋 |
| 失敗 (`failed`) | 🔴 | 處理失敗，可重新觸發 |
| 已封存 (`archived`) | ⚫ | 已封存，不參與搜尋 |

## 建立 Wiki 頁面

### 新增頁面

1. 在 Wiki 頁面右上角，點擊 **「+ 新增頁面」** 按鈕
2. 系統會建立一個新的草稿頁面
3. 填寫頁面標題，開始撰寫內容（支援 Markdown 格式）
4. 完成後點擊 **「發佈」** 讓其他成員可以看到

### 頁面範圍（Scope）

建立頁面時，您需要選擇頁面的可見範圍：

| 範圍 | 說明 | 誰可以看到 |
|------|------|------------|
| **組織** (organization) | 企業級頁面，對所有成員可見 | 所有組織成員 |
| **工作區** (workspace) | 工作區專屬頁面 | 該工作區成員 |
| **私人** (private) | 個人筆記，僅自己可見 | 僅建立者 |

### 編輯頁面

1. 點擊頁面標題或 **「編輯」** 按鈕
2. 直接在編輯器中修改內容
3. 修改會自動儲存為草稿，確認後點擊 **「更新」**

### 封存頁面

1. 在頁面右上角點擊 **「⋯」** 選單
2. 選擇 **「封存」**
3. 封存後頁面移至側邊欄的 **「封存」** 區塊，不再出現在主清單

### 還原封存頁面

1. 展開側邊欄的 **「封存」** 區塊
2. 找到要還原的頁面，點擊 **「⋯」** 選單
3. 選擇 **「還原」**

## 上傳知識文件

知識文件（PDF、DOCX、HTML 等）可透過以下路徑上傳：

1. 前往對應**工作區**的 **「知識」** 分頁
2. 點擊 **「上傳文件」** 按鈕
3. 選擇檔案（支援 PDF / DOCX / DOC / HTML / TXT / MD）
4. 填寫文件元資料（以下說明各欄位）
5. 點擊 **「上傳」**

上傳後，系統會自動進行 AI 索引流程（約 1-10 分鐘，視文件大小而定）。完成後文件狀態變為 **「已就緒」**，即可在 Wiki 知識搜尋中查詢。

### 上傳限制

| 項目 | 限制 |
|------|------|
| 支援格式 | PDF、DOCX、DOC、HTML、HTM、TXT、MD |
| 最大檔案大小 | **50 MB** |
| 最小檔案大小 | 1 KB |
| 檔案名稱長度 | 最多 255 字元 |

### 元資料欄位說明

#### 分類（Taxonomy）⭐ 必填

Taxonomy 是文件的語意分類，AI 系統會根據此分類進行向量搜尋過濾，影響搜尋結果的準確性。

| 分類值 | 適用文件 |
|--------|---------|
| `规章制度` | 員工手冊、考勤辦法、薪資規定、HR 合規文件 |
| `技術文件` | 系統架構圖、API 規格、ADR、技術規範 |
| `產品手冊` | 功能說明書、產品規格、版本發布說明 |
| `操作指南` | SOP 作業流程、How-to 教學、部署指南 |
| `政策文件` | 資安政策、個資聲明、合規政策 |
| `訓練教材` | 新人訓練簡報、技術教學資料 |
| `研究報告` | 市場研究、競品分析、使用者調查報告 |
| `其他` | 以上皆不適用時使用 |

> 💡 **選對分類很重要！** 當使用者搜尋「規章制度相關問題」時，系統會優先搜尋 taxonomy = `规章制度` 的文件，可大幅提升回答準確度。

#### 部門（Department）選填

填寫文件所屬部門，例如：`人資部`、`工程部`、`產品部`。

#### 語言（Language）選填

文件的主要語言，ISO 639-1 格式。預設為 `zh-TW`（繁體中文）。
常見值：`zh-TW`、`zh-CN`、`en`、`ja`。

#### 標籤（Tags）選填

自訂關鍵字標籤，用於補充分類之外的語意，例如：`2024年度`、`重要`、`Q3`。

#### 存取控制（Access Control）選填

控制哪些角色的使用者可以在 AI 搜尋中看到此文件。
預設值：`["Admin", "Member"]`（組織所有成員均可搜尋）。

## 常見問題

### AI 搜尋沒有找到我要的資料？

可能原因：
- 文件仍在 **「處理中」** 狀態，尚未完成索引 → 等待處理完成
- 文件您沒有存取權限 → 聯絡工作區管理員確認 `accessControl` 設定
- 問題措辭與文件內容差距較大 → 嘗試換個說法或用更具體的關鍵字

### 文件一直卡在「處理中」？

若超過 15 分鐘仍在 **「處理中」**，可能是 AI 處理失敗。請：
1. 前往工作區知識頁面，查看文件是否轉為 **「失敗」** 狀態
2. 若為失敗，點擊文件列的 **「重試」** 按鈕
3. 若問題持續，聯絡系統管理員

### 如何讓組織所有人都能搜尋到某份文件？

上傳文件時，將 `accessControl`（存取控制）設定為 `["Admin", "Member"]`，讓所有組織成員都能在 AI 搜尋中找到該文件。

### Wiki 頁面和知識文件有什麼不同？

| | Wiki 頁面 | 知識文件 |
|--|----------|---------|
| 建立方式 | 直接在 Wiki 介面撰寫 | 上傳外部檔案（PDF/DOCX 等） |
| 格式 | Markdown 文字 | 原始檔案 |
| AI 搜尋 | 目前為手動瀏覽（未來支援） | ✅ 已支援 RAG 語意搜尋 |
| 適合用途 | 操作說明、政策說明、記錄 | 正式文件、規章制度、技術手冊 |
