# 排程功能使用手冊

> **文件版本**：v1.0.0
> **最後更新**：2026-03-20
> **目標讀者**：工作區擁有者、組織管理員、組織成員

---

## 概覽

排程功能採用**雙向請求-履行模型**：

- **工作區端（Demand 側）**：工作區擁有者提出人力需求（資源請求）
- **組織端（Supply 側）**：組織管理員審核需求並指派適合的成員
- **可見性**：雙方均可即時查看請求的最新狀態

---

## 工作區端使用指引

### 進入排程頁面

1. 登入後，從側邊欄選擇您的工作區
2. 在工作區詳細頁面，點擊 **「排程」** 分頁
3. 您將看到 **「資源請求」** 介面

### 提交新的資源請求

1. 點擊右上角的 **「+ 新增請求」** 按鈕，展開新增表單

2. 填寫表單欄位：

   | 欄位 | 必填 | 說明 |
   |------|------|------|
   | **需求說明** | ✅ | 描述人力需求的背景與工作內容（例如：「需要一位熟悉 React 的前端工程師支援三個月的專案開發」） |
   | **期望開始時間** | ❌ | 選擇期望的開始日期與時間（可留空，由組織安排） |

3. 確認內容後，點擊 **「提交」** 按鈕

4. 提交成功後，新請求會立即出現在下方清單中，狀態為 **「已提交」**

> **提示**：需求說明越詳細，組織管理員越能快速媒合適合的成員。

### 查看請求狀態

提交後，您可在排程頁面的請求清單中查看每項請求的最新狀態：

| 狀態 | 說明 | 您需要做什麼 |
|------|------|--------------|
| 🔵 **已提交** | 請求已送出，等待組織審核 | 等待即可 |
| 🟡 **審查中** | 組織正在評估您的需求 | 等待即可 |
| 🟢 **已接受** | 組織已接受請求，正在準備分派 | 等待分派通知 |
| 🔴 **已拒絕** | 組織拒絕了請求 | 查看拒絕原因，修改後重新提交 |
| ⚫ **已取消** | 請求已被取消 | 如需要可重新提交 |
| ⚫ **已結束** | 請求已完成或關閉 | — |

每項請求也會顯示**最後更新時間**，以及**拒絕或取消原因**（如有）。

### 取消請求

對於狀態為 **「已提交」** 或 **「草稿」** 的請求，**原提交者本人**可點擊請求行右側的 **✕** 圖示來取消。取消後，狀態會立即更新為 **「已取消」**，且該請求會自動從組織端的「待分派」清單中移除。

## 組織端使用指引

### 進入排程管理頁面

1. 切換至組織帳戶（從頂部帳戶選擇器）
2. 從側邊欄選擇 **「排程」**
3. 您將看到組織排程管理介面

### 查看待分派請求（列表視圖）

預設顯示的是**列表視圖**，分為兩個主要區塊：

#### 待分派（資源請求）

顯示所有來自各工作區、狀態為 **「已提交」** 的資源請求。每筆記錄包含：

- 工作區來源
- 請求狀態徽章
- 已指派成員（如有）
- 最後更新時間與原因

#### 排程項目

顯示已確認的排程項目清單。您可依以下狀態頁籤篩選：

| 頁籤 | 說明 |
|------|------|
| **即將到來** | 未來的排程項目 |
| **未確認** | 等待確認的項目 |
| **定期** | 週期性重複排程 |
| **之前** | 過去的排程記錄 |
| **已取消** | 已取消的項目 |

### 查看月曆視圖

點擊右上角的 📅 圖示切換到**月曆週視圖**：

- 以 **GMT+8** 時區顯示
- 橫向為 **週日至週六**，縱向為 **00:00 至 23:00**（24 小時）
- 今天的日期以圓圈標示
- 點擊 **◀ ▶** 箭頭切換週次
- 排程項目以色塊顯示在對應的時段格子

> 月曆視圖設計參考了 Postiz 社群媒體排程平台的週視圖模式，提供直觀的時間軸管理體驗。

### 分派請求（規劃中）

完整的分派流程（審核 → 任務分解 → 候選人比對 → 指派 → 排程確認）目前為系統開發中，預計在後續版本提供以下操作：

1. 審核資源請求（接受 / 拒絕）
2. 系統自動比對符合條件的組織成員
3. 管理員確認分派對象
4. 系統通知被分派成員
5. 成員接受後，系統確認排程時段

## 請求生命週期圖

```
工作區提交
    │
    ▼
 已提交 (submitted)
    │
    ▼
 審查中 (under-review)
    │
    ├──── 拒絕 ────→ 已拒絕 (rejected) ──→ 已結束 (closed)
    │
    ▼
 已接受 (accepted)
    │
    ├──── 系統比對 ────→ 任務建立 ──→ 比對 ──→ 分派 ──→ 排程確認
    │
    └──── 已結束 (closed)
```

## 常見問題

### Q：我提交了請求，但清單沒有出現？

**A**：請檢查以下項目：
1. 確認「需求說明」欄位已填寫（必填）
2. 點擊「提交」後，等待 1-2 秒讓頁面刷新
3. 若仍未出現，請重新整理頁面（F5）
4. 若問題持續，請聯繫系統管理員

### Q：我的請求被拒絕了，原因在哪裡？

**A**：在請求清單中，每筆被拒絕的請求下方會顯示拒絕原因。若原因顯示為 `not_approved`，表示未提供具體原因，建議聯繫組織管理員確認細節。

### Q：我可以修改已提交的請求嗎？

**A**：目前無法直接修改已提交的請求。若需變更，請取消現有請求並重新提交。取消功能僅適用於「已提交」或「草稿」狀態的請求。

### Q：月曆視圖顯示的是哪些項目？

**A**：月曆視圖顯示的是**已排程的工作區項目**（`WorkspaceScheduleItem`），包含里程碑、跟進事項和維護任務。資源請求在完成分派並確認時段後，才會以排程項目的形式出現在月曆中。

### Q：「待分派」區塊中看到其他工作區的請求嗎？

**A**：是的，「待分派」區塊聚合了組織內**所有工作區**的待處理資源請求，方便組織管理員統一審核與安排。

## 狀態顏色對映

本系統使用統一的狀態顏色語意：

| 顏色 | 狀態 | 語意 |
|------|------|------|
| 🔲 白底邊框 | 草稿、已取消、已結束 | 非活躍狀態 |
| 🔵 灰底 | 已提交 | 等待處理 |
| 🔵 藍底 | 審查中、已接受 | 進行中 |
| 🔴 紅底 | 已拒絕 | 需要關注 |
