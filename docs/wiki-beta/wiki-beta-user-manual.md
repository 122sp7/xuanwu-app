## 2. 第一次使用：五分鐘快速體驗

本節帶你完成「上傳文件 → 等待解析 → 執行 RAG 問答」的最短路徑。

### 步驟 1：進入 Wiki-Beta

1. 點選左側 App Rail 中的 **Wiki Beta**。
2. 預設進入 **Documents** 頁面。

> **你會看到**：空的文件列表（若尚未上傳任何文件）或既有文件，以及頁面上方的上傳區塊。

### 步驟 2：上傳文件

1. 將 `.pdf`、`.tiff`、`.png` 或 `.jpeg` 檔案**拖入 Drop Zone**，或點擊該區域選擇檔案。
2. 確認畫面出現檔案名稱後，點選 **「上傳並啟動解析」**。
3. 上傳按鈕顯示 loading 狀態，完成後文件出現在列表中，status 為 `processing`。

> **你會看到**：文件列表新增一筆記錄，`status` 欄顯示 `processing`，`rag status` 顯示 `pending`。背景解析通常需要 30 秒至數分鐘，視文件大小而定。

### 步驟 3：等待解析完成

1. 在 Documents 列表觀察 `status` 欄位：
   - `processing` → 解析中，請稍候
   - `ready` → 解析完成，可進行 RAG 問答
   - `error` → 解析失敗，請參考常見問題 Q2
2. 點選 **「刷新文件」** 按鈕，或重新整理頁面，查看最新狀態。

### 步驟 4：執行 RAG 問答

1. 從側邊欄進入 **RAG Query**（或直接訪問 `/wiki-beta/rag-query`）。
2. 在輸入框輸入你的問題（例如：「這份文件的主要結論是什麼？」）。
3. `top_k` 保留預設值即可。
4. 點選 **「送出查詢」**。
5. 查看 **Answer**（AI 回答）與 **Citations**（引用來源）。

> **你會看到**：Answer 區塊顯示 AI 生成的回答，Citations 區塊列出原始文件片段與出處。若無引用，請參考常見問題 Q3。

## 3. 核心操作說明

### 3.1 查看文件（Documents）

**預設行為**：Documents 頁面顯示目前帳號下**所有工作區**的文件，避免因切換工作區而誤以為文件遺失。

**workspace 篩選**：若只想看特定工作區的文件，在 URL 加入 `?workspaceId=<id>` 參數。

**文件欄位說明**：

| 欄位 | 說明 |
|---|---|
| filename | 原始上傳檔名 |
| status | 解析狀態：`processing` / `ready` / `error` |
| rag status | RAG 索引狀態：`pending` / `indexed` / `error` |
| pages | 文件頁數（解析完成後顯示） |
| uploadedAt | 上傳時間 |

### 3.2 觸發手動重整（RAG Reindex）

當文件的 RAG 索引需要重新建立時（例如向量庫更新後），可手動觸發重整：

1. 進入 **RAG Reindex**（`/wiki-beta/rag-reindex`）或在 Documents 列表中操作。
2. 找到目標文件，點選 **「手動重整」**。
3. 按鈕顯示 loading 狀態，完成後 `rag status` 更新。

> **注意**：「手動重整」按鈕在文件尚未完成解析（無 json_gcs_uri）時為灰色不可按。請先確認 `status` 為 `ready`。

## 4. 常見問題

### Q1：我剛上傳的文件，為什麼在列表看不到？

**原因**：文件可能仍在背景處理，或你的 workspace 篩選過濾掉了它。

**請按順序確認**：
1. 點選 **「刷新文件」** 或重新整理頁面。
2. 確認目前 account 是否正確（App Rail 右上角）。
3. 如果有 workspace 篩選參數，先移除改看全覽（移除 URL 中 `workspaceId` 參數）。
4. 文件 `status` 若仍是 `processing`，等待 1–2 分鐘後再刷新。

### Q2：為什麼文件狀態顯示 `error`？

**原因**：解析流程遭遇問題（格式不支援、檔案損毀、後端服務暫時無法使用）。

**請按順序確認**：
1. 確認檔案格式為 `.pdf`、`.tiff`、`.png` 或 `.jpeg`。
2. 確認檔案可正常開啟，未加密或損毀。
3. 嘗試重新上傳同一份檔案。
4. 若問題持續，聯繫運維人員提供 `documentId`（列表中可見）以供後端診斷。

### Q3：RAG Query 送出後，答案沒有 Citations（引用）？

**原因**：文件的向量索引尚未建立，或問題語意與現有文件差距過大。

**請按順序確認**：
1. 到 Documents 確認相關文件的 `rag status` 為 `indexed`（不是 `pending` 或 `error`）。
2. 若 `rag status` 為 `pending`，等待索引完成後再試。
3. 若 `rag status` 為 `error`，對該文件執行 **「手動重整」** 後再試。
4. 嘗試換更具體的問題，確保問題語意與文件內容相關。

### Q4：為什麼無法點選「上傳並啟動解析」？

**原因**：缺少必要條件。按鈕在以下情況為不可用（disabled）：
- 未選擇任何檔案
- 目前帳號無效或未登入

**解決方式**：確認已選擇檔案，且帳號登入狀態正常。

## 5. 操作建議

- **日常觀察**：每天先在 Documents 瀏覽 `status` / `rag status`，確認文件管道正常運作。
- **先重整再查詢**：有新文件或文件內容變更時，先執行「手動重整」確保向量索引是最新的，再進行 RAG 問答。
- **跨工作區稽核**：需要確認某份文件是否存在時，先用「account 全覽」（不帶 workspace 篩選）搜尋，再確認所屬工作區。

## 6. 資料範圍與權限說明

| 概念 | 說明 |
|---|---|
| account（帳號） | 資料主範圍。你上傳的所有文件都屬於你的帳號，不與其他帳號共用。 |
| workspace（工作區） | 同一帳號下的視角分組。workspace 是篩選用途，不是資料隔離邊界。 |
| Namespace | 背景路由能力，不在 UI 中獨立暴露，使用者無需主動操作。 |

> 你只能看到自己帳號下的資料。切換工作區不會讓你看到其他帳號的文件。
