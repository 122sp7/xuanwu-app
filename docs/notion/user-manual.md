## 2. 功能地圖

Xuanwu 目前 Notion 對齊的功能位於 **Wiki-Beta** 模組下：

| 功能 | 對應 Notion 概念 | 路徑 |
|---|---|---|
| Pages | Page tree（階層頁面） | `/wiki-beta/pages` |
| Libraries | Database / Data source（結構化資料表） | `/wiki-beta/libraries` |
| Documents | Knowledge documents（知識文件庫） | `/wiki-beta/documents` |
| RAG 問答 | AI 知識查詢 | `/wiki-beta/rag-query` |

## 3. Pages 使用指南

### 3.1 什麼是 Pages？

Pages 對應 Notion 的 Page tree，讓您可以用階層結構組織知識內容。每個頁面可以有多個子頁面，形成樹狀結構。

### 3.2 進入 Pages

1. 從左側 App Rail 點選 **Wiki Beta**。
2. 在 Wiki-Beta 導覽列選擇 **Pages**。
3. 您將看到目前帳戶下所有根頁面的樹狀列表。

### 3.3 建立新頁面

**建立根頁面（Root Page）**：

1. 進入 `/wiki-beta/pages`。
2. 點選頁面右上角的 **新增頁面** 按鈕。
3. 輸入頁面標題（最長 120 字元）。
4. 點選 **建立** 完成。

**建立子頁面（Sub-page）**：

1. 在頁面樹中找到目標父頁面。
2. 點選該頁面右側的 **+** 圖示。
3. 輸入子頁面標題。
4. 點選 **建立** 完成。

> 子頁面自動歸屬於所選的父頁面，並繼承相同的帳戶範圍（account scope）。

### 3.4 更名頁面

1. 在頁面樹中找到目標頁面。
2. 點選頁面名稱旁的 **編輯（鉛筆）** 圖示，或雙擊頁面名稱。
3. 輸入新標題。
4. 按下 Enter 或點選 **儲存** 完成。

### 3.5 移動頁面

1. 在頁面樹中找到要移動的頁面。
2. 拖曳頁面到目標位置，或點選 **移動** 選項。
3. 選擇新的父頁面（或選擇「移至最上層」升為根頁面）。
4. 確認移動。

> **注意**：系統會自動阻擋形成循環的移動（例如把父頁面移動到其子頁面下）。

### 3.6 常見問題

**Q：為何看不到我建立的頁面？**
- 確認目前 account 是否正確。
- 確認頁面狀態是否為 `active`（封存的頁面不顯示在樹中）。

**Q：為何不能將頁面移到某個位置？**
- 該操作可能造成循環結構（頁面成為自己的子孫），系統會自動阻擋。

## 4. Libraries 使用指南

### 4.1 什麼是 Libraries？

Libraries 對應 Notion 的 Database（資料庫），讓您可以用結構化的欄位和列（Records）來管理資料，例如任務清單、文件索引、技能清冊等。

### 4.2 進入 Libraries

1. 從左側 App Rail 點選 **Wiki Beta**。
2. 在 Wiki-Beta 導覽列選擇 **Libraries**。
3. 您將看到目前帳戶下所有 Libraries 的列表。

### 4.3 建立新 Library

1. 進入 `/wiki-beta/libraries`。
2. 點選 **新增資料庫** 按鈕。
3. 輸入 Library 名稱（最長 80 字元）。
4. 點選 **建立** 完成。

### 4.4 新增欄位（Fields）

Library 建立後預設無欄位。您需要先新增欄位才能輸入記錄。

支援的欄位型別：

| 型別 | 說明 | 範例用途 |
|---|---|---|
| `title` | 標題文字 | 任務名稱、文件標題 |
| `text` | 多行文字 | 說明、備註 |
| `number` | 數字 | 優先級、分數 |
| `select` | 單選清單 | 狀態、類別 |
| `relation` | 跨紀錄引用 | 關聯頁面或其他記錄 |

**新增欄位步驟**：

1. 進入目標 Library。
2. 點選 **新增欄位（+）**。
3. 輸入欄位標籤（Label）和鍵名（Key）。
4. 選擇欄位型別。
5. 若為 `select` 型別，輸入可選項目清單。
6. 勾選是否為必填欄位（Required）。
7. 點選 **儲存** 完成。

### 4.5 新增記錄（Records）

記錄對應 Notion 資料庫中的一列（Row）。

1. 進入目標 Library。
2. 點選 **新增記錄（+）** 或 **New Record**。
3. 填入各欄位的值（必填欄位不可留空）。
4. 點選 **建立** 完成。

### 4.6 常見問題

**Q：為何無法建立記錄？**
- 確認所有標示為「必填」的欄位都已填入值。
- 確認 Library 已建立至少一個欄位。

**Q：為何 `select` 欄位沒有選項？**
- 建立或編輯欄位時，需在 `options` 欄位輸入可選項目。

## 5. Documents 快速建立（Notion-like 入口，planned）

### 5.1 功能說明

Documents 側邊欄右側的 **`+`** 圖示提供快速建立頁面或資料庫的入口，對齊 Notion 在側邊欄快速新增內容的使用體驗。

> 本功能為規劃中（planned），尚未正式上線。

### 5.2 操作流程（上線後）

1. 在 Wiki-Beta 側邊欄找到 **Documents** 項目。
2. 點選 Documents 右側的 **`+`** 圖示。
3. 在彈出選單選擇：
   - **新增頁面**：建立新的階層頁面
   - **新增資料庫**：建立新的 Library
4. 填入名稱後確認建立。
5. 系統顯示成功提示並導向新建項目。

## 6. RAG 知識問答

RAG（Retrieval-Augmented Generation）讓您可以對已上傳的知識文件進行自然語言問答。

### 6.1 上傳知識文件

1. 進入 `/wiki-beta/documents`。
2. 將檔案拖入 Upload 區塊，或點擊選檔。
3. 確認格式為 `.pdf`、`.tiff`、`.png`、`.jpg`。
4. 點選 **上傳並啟動解析**。
5. 等待背景 pipeline 完成（status 從 `processing` 變為 `ready`）。

### 6.2 進行 RAG 問答

1. 進入 `/wiki-beta/rag-query`。
2. 在輸入框輸入問題。
3. 設定 `top_k`（決定參考幾筆相關片段，可留預設值）。
4. 點選 **送出查詢**。
5. 查看 **Answer**（答案）與 **Citations**（引用來源）。

## 7. 操作建議

- **日常知識管理**：使用 Pages 組織層級結構，將相關頁面建立為子頁面。
- **任務與資料管理**：使用 Libraries 建立結構化清單（任務、文件索引、技能清冊等）。
- **AI 知識查詢**：先在 Documents 確認文件狀態為 `ready` 後，再進行 RAG 問答。
- **workspace 視角**：若需查看特定工作區內容，使用 `workspaceId` 篩選；若需全帳戶瀏覽，清除篩選即可。

---

## 8. 權限與資料範圍

- 所有 Pages、Libraries、Documents 均以 **account** 為資料主範圍。
- workspace 是同帳戶下的視角篩選，不代表獨立的資料邊界。
- 您只能存取自己帳戶下的資料；跨帳戶資料不可見。
